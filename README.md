# 本日の予定ビューア

Mac の Calendar.app に表示されている全カレンダー（Google 同期の「メモ」等も含む）から、
**今日の予定を時間順に取得して表示する**ローカルスクリプト。

`.command` をダブルクリックすると、こんな形で出力されます。

```
おはようございます

【本日の予定】
・スケジュール確認 プラネットくん
・全体会議
```

- OAuth / API キー不要（Calendar.app から直接読む）
- 実行はローカルのみ。Claude のトークンは消費しない
- 予定がない日は `・本日の予定はありません` と表示

---

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `today_schedule.py` | 本体（カレンダー取得・整形ロジック） |
| `本日の予定.command` | ダブルクリック用ランチャー（venv を用意して .py を実行） |
| `README.md` | このファイル |

このフォルダごと `~/Obsidian/Project/` に置く前提。
**フォルダの中に入るのはこの 3 ファイルだけ**。Python 環境（venv）は後述のとおり
vault の外に作られるので、Obsidian Sync / iCloud に巻き込まれない。

---

## セットアップ（初回のみ）

1. 3 ファイルを同じフォルダに置く（`~/Obsidian/Project/DailyScheduleViewer/` など）。

2. ランチャーに実行権限を付ける。

   ```bash
   chmod +x "本日の予定.command"
   ```

3. `本日の予定.command` をダブルクリック。
   - **初回だけ** 専用の Python 環境（venv）を自動作成し、`pyobjc` を入れる
     （数十秒〜1 分ほどかかる）。手動の `pip install` は不要。
   - 続けて macOS が「カレンダーへのアクセス」を確認するので**許可**する。

以降はダブルクリックするだけで即表示される。

> デスクトップから一発で開きたい場合は、`.command` の**エイリアス**を
> デスクトップに置く（実体はこのフォルダのまま）。

---

## Python 環境（venv）について

- venv は **vault の外**、`~/.venvs/daily-schedule-viewer/` に作られる。
- そのため vault 内には**コードだけ**が残り、同期が荒れない
  （venv は小ファイルの塊で、iCloud / Obsidian Sync と相性が悪いため外に逃がしている）。
- activate は不要。`.command` が venv の python を直接呼んで実行する。
- **環境をリセットしたいとき**は次を実行すれば、次回ダブルクリック時に作り直される。

  ```bash
  rm -rf ~/.venvs/daily-schedule-viewer
  ```

> ホーム直下の `~/venv`（昔作った汎用 venv）とは無関係。競合しない。

## 使い方

- `本日の予定.command` をダブルクリックするだけ。
- ターミナルが開いて結果を表示し、`[Process completed]` で止まる。閉じて OK。

## カスタマイズ

`today_schedule.py` の冒頭にある定数を編集する。

| 定数 | 既定 | 説明 |
| --- | --- | --- |
| `GREETING` | `おはようございます` | 先頭の挨拶 |
| `HEADER` | `【本日の予定】` | 見出し |
| `NO_EVENTS_LINE` | `・本日の予定はありません` | 予定ゼロ時の表示 |
| `SHOW_TIME` | `False` | `True` で各行頭に開始時刻（例 `・13:00 …`）を付ける |

### 対象カレンダーについて

Calendar.app で**表示（チェック）がオンになっている全カレンダー**が対象。
特定のカレンダーだけ含めたい／除外したい場合は `fetch_today_events()` 内の
`store.calendarsForEntityType_(...)` の結果をフィルタすればよい
（Claude Code に「○○カレンダーだけ対象に」と頼めば直してくれる）。

---

## 改修の進め方

このフォルダを **Claude Code** で開けば、編集・実行・改良をその場で回せる。
依頼例:

- 「各予定に開始〜終了時刻を付けて」
- 「終日予定はリストの末尾にまとめて」
- 「明日の予定も出せるオプションを追加して」

ロジックを変えるだけなら venv は作り直し不要。新しいパッケージが必要になった場合だけ、
`~/.venvs/daily-schedule-viewer/bin/python3 -m pip install ○○` で追加する
（または venv を削除して `.command` に追記 → 次回自動再構築）。

## トラブルシュート

| 症状 | 対処 |
| --- | --- |
| `カレンダーへのアクセスが許可されていません` | システム設定 > プライバシーとセキュリティ > カレンダー で、実行元（ターミナル等）を許可 |
| ダブルクリックで開かない | `chmod +x "本日の予定.command"` を実行 |
| 初回セットアップが失敗する | ネット接続を確認 → `rm -rf ~/.venvs/daily-schedule-viewer` してから再度ダブルクリック |
| 予定が一部出ない | Calendar.app でそのカレンダーの表示がオンか確認 |
